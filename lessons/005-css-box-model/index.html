box-sizing: content-box

The following <div> element uses the browser default value "content-box" for the "box-sizing" property and the full width of the box is calculated by adding all margins, paddings and borders to the "width" property.

The used styles:
border: 6px solid #000000
height: 130px
margin: 20px
padding: 20px
width: 620px
Full width calculation:

margin-left + border-left + padding-left + width + padding-right + border-right + margin-right

Result:

20px + 6px + 20px + 620px + 20px + 6px + 20px = 712px

box-sizing: border-box

The following <div> element uses the newer value "border-box" for the "box-sizing" property and the full width of the box is fixed set by the "width" property.

Other than with "content-box" the inner width of the box is reduced by all paddings and borders.

The used styles:
border: 6px solid #000000
height: 360px
margin: 20px
padding: 20px
width: 620px
Full width calculation:

margin-left + width + margin-right

Result:

20px + 620px + 20px = 660px


Inner width calculation:

width - padding-left - border-left - padding-right - border-right

Result:

620px -20px -6px -20px -6px = 568px

Floating elements

Float: left

This is a text with an image floating to the left, so that the text is floating around on the right side of the image element.

This can be useful for blog articles, magazines, newspapers and lots of other use cases, where you'd like show an image and have more text that should be floating around it on one side.

The text will continue below the image, as soon as it needs more space.

The text will continue below the image, as soon as it needs more space.

Using a clearfix

One of those pitfalls is that occasionally the proper styles will not render on an element that it is sitting next to or is a parent element of a floated element. When an element is floated, it is taken out of the normal flow of the page, and, as a result, the styles of elements around that floated element can be negatively impacted.

It is possible to clear the floating manually, by using "clear: both;" on a following element, etc. but there is a new and more comfortable solution today.

It is common to use a special class on the element that contains multiple other floating elements. If there is none, you would probably use a <div> element as a solution.

The surrounding element then get's the class "clearfix" added to it, that will care about clearing all floats inside the box, so that the elements inside will not affect the parent element's styling.

The details how the clearfix works, are discusses later.
